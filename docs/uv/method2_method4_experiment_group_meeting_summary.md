# Method2 / Method4 实验链路组会总结

## 1. 背景与目标

这组实验的核心目标，是把一个反复表现不稳定的 UV 迁移链路拆开看清楚：问题到底出在上游构场、下游求解、低模自由度，还是这几者之间的错配。

我们并不是一开始就在验证某个单独的改法，而是先沿着“哪里先出问题”这条线，把 Method2 和 Method4 逐层解剖。整个实验链是递进的：先排除明显的工程性嫌疑，再审计目标场本身，接着验证求解器是否背锅，最后才进入场空间投影、求解空间约束、拓扑释放和 Method2.5 组合验证。

## 2. Method2 的方法原理

Method2 的基本想法，不是直接把高模的 UV 坐标拷到低模上，而是先把高模局部的纹理变化方式转成面级的局部微分信息，再让低模去拟合这组局部变化。

它的主链路可以概括为以下几个阶段：

1. 先在高模上建立 UV 连续域的分区。
2. 把这个分区的语义转移到低模面片上。
3. 沿这些分区边界在低模上切开拓扑，把低模变成可分块求解的 chart。
4. 在低模面内做采样，通过射线命中和最近点回退，在高模上找到对应位置。
5. 把高模局部的 UV Jacobian 收集回来，并在低模每个面上做稳健聚合。
6. 把这些面级目标 Jacobian 变成边差分约束，再加上平滑项、软锚点和数值稳定项。
7. 解一个稀疏的线性泊松系统，恢复低模顶点的 UV。

这条链路的优势，是每一层都有明确的几何解释，也有工程保护：采样失败可以回退，分岛可以拆解求解，锚点可以防漂移，求解器也有后备链。

它的核心风险也很明确：只要上游构造出来的目标 Jacobian 场不够自洽，后面的线性积分器就只能在互相冲突的局部要求之间做折中，而这个折中很容易表现成拉伸、翻转或者越界。

## 3. Method4 的方法原理

Method4 不是一条独立的建图方法，它完全依赖 Method2 先把对应关系、面级目标场、初始 UV 和锚点准备好。

它的作用，是拿着 Method2 给出的初始化和目标 Jacobian，再做一轮非线性的 refinement。这个 refinement 不再只看“当前梯度像不像目标梯度”，还会显式加入几何保护项，例如：

1. 抑制过度畸变的对称狄利克雷项。
2. 鼓励正面积的对数行列式项。
3. 阻止翻转的 barrier 项。
4. 维持整体稳定的软锚点约束。

因此 Method4 的定位，不是重建语义，也不是重新采样，而是“在 Method2 已经给出一个大致方向之后，尽量把这个 UV 解往更可行的几何形态拉”。

它的前提也很强：如果 Method2 交给它的初始化太脏、翻转太重、目标场本身太冲突，那么 Method4 也可能根本迈不出有效的一步。

## 4. 整体实验推进逻辑

这十几个实验并不是平行罗列的，而是围绕一个不断收紧的问题链条展开：

1. 先排除是不是小碎块、无效面、分岛方式这些显眼工程因素导致的。
2. 再直接审计目标场本身是否存在明显旋度和不闭合。
3. 接着验证局部可不可以解、全局为什么会坏。
4. 然后拆解目标场的来源，检查 fallback、贡献者混合、锚点底图和残差之间的关系。
5. 最后进入场空间投影，尝试在送入求解器之前先把“有毒的场”清洗掉。

## 5. 实验 1 到 13 的过程总结

### 5.1 实验 1：过滤指标实验

这个实验的目标，是先排除一个朴素的怀疑：坏结果是不是主要由无效面、极小组件或者几乎没有样本支持的小碎块造成的。

实现逻辑是，把同一份结果按不同过滤条件切开看，例如只看有效面、只看有采样支持的面、只看足够大的组件，重新统计 UV 质量和 Jacobian 误差。

结果表明，过滤掉明显糟糕的边角数据以后，整体问题并没有本质消失，大组件和有效面内部依然存在显著的畸变。

初步结论是，问题不是“少数垃圾区域拖垮了统计”，而是主干区域本身也有系统性冲突。

### 5.2 实验 2：旋度审计

第二个实验的目标，是检查目标场本身是否可积。

实现逻辑是，从相邻面之间的边跳变出发，统计目标 Jacobian 在边界处的不连续程度，再沿顶点环和面环计算闭合残差，看这个场是否存在明显的局部旋度。

结果非常明确：目标场里确实存在成片的边跳变和环残差，高值区域不是零星噪声，而是能形成统计意义上的高分位尾部。

初步结论是，Method2 当前构造出来的目标场，并不是一个天然适合被泊松积分的平滑场。

### 5.3 实验 3：重分岛与重分块实验

既然旋度已经被观察到了，第三个实验要回答的是：这些冲突是不是因为当前的岛划分方式不对。

实现逻辑是，尝试多种重分块方式，包括沿高跳变边切开、沿高协方差边切开、直接按有效连通块重划，甚至退化到全局求解。

结果是，无论是多切一点、少切一点，还是换一种连通定义，指标都没有出现决定性的翻转。

初步结论是，分岛策略会影响局部表现，但它不是当前失败的主导原因。真正更像根病的，仍然是更上游的目标场本身。

### 5.4 实验 4：局部 patch 狄利克雷实验

第四个实验的目标，是判断全局解坏掉，到底是因为局部块本身不可解，还是因为全局拼装时冲突累积。

实现逻辑是，从高误差面、高方差面和相对健康的控制面各选一些种子，在局部 patch 上把边界 UV 固定住，只解内部的小系统。

结果是，局部 patch 基本都能解出来，而且很多 patch 的拉伸和 Jacobian 误差都远好于全局结果。

初步结论是，问题并不是“每个局部块都已经胡了”，而更像是“局部看上去还能解释，但一拼成全局就开始累积冲突”。

### 5.5 实验 5：贡献者混合审计

第五个实验的目标，是检查一个低模面所接收到的高模样本，是不是本身就来自互相不兼容的来源。

实现逻辑是，按最终误差高低和方差高低把面分层，统计这些面各自接收了多少不同的高模面、多少不同的高模岛、多少 fallback 样本，以及这些样本在 Jacobian 空间里有多分散。

结果显示，问题面普遍具有更强的多源混合、更高的样本分散和更高的 fallback 参与比例。

初步结论是，Builder 很可能正在把互相打架的样本强行搅成一个面级目标场。

### 5.6 实验 6：前后对齐实验

第六个实验的目标，是判断 Method2 的后处理平移校正步骤是不是在破坏原本的结果。

实现逻辑是，把线性积分后的 UV 在平移校正前后分别做质量评估、Jacobian 对比和采样残差统计。

结果是，后对齐对重投影和越界有一定修补作用，但对拉伸和 Jacobian 误差几乎不改变大局。

初步结论是，后对齐只是症状级修补，不是根病来源，也不可能单独把问题治好。

### 5.7 实验 7：精确场回灌实验

第七个实验的目标，是给求解器一次“公正的考试”，判断只要输入场本身自洽，Method2 的线性积分器到底能不能工作。

实现逻辑是，把已经求出来的 UV 重新转成精确的面级目标场，再把这组场喂回求解器；同时也把由锚点投影得到的顶点 UV 转成精确场重新回灌。

结果是，只要输入场来自一个真实、连续的 UV 标量场，求解器就能以非常小的误差把它恢复回来。

初步结论是，线性积分器不是先天坏掉，而是平时喂进去的目标场本来就不干净。

### 5.8 实验 8：当前目标场与锚点底图对比

第八个实验的目标，是判断高误差区域是不是正好也是“当前目标场远离锚点底图”的区域。

实现逻辑是，先用锚点恢复一版完整顶点 UV，再求出它在低模上的面级 Jacobian，把这版底图和当前目标场逐面对比。

结果表明，高误差面、高方差面，往往恰好也是当前目标场偏离锚点底图最严重的面。

初步结论是，锚点底图是相对健康的，而当前目标场在某些地方明显跑偏了。

### 5.9 实验 9：fallback 消融实验

第九个实验的目标，是判断 fallback 样本到底是在救 coverage，还是在向目标场里持续注毒。

实现逻辑是，做三种版本：保留当前权重、完全移除 fallback 样本、保留 fallback 但显著降权，然后重新聚合面级场并重新求解。

结果表明，完全移除 fallback 以后，局部场确实更纯了，但 coverage 明显下降；保留 fallback 但降权，有时能减轻污染，但又不足以彻底解决问题。

初步结论是，fallback 是一杯毒酒，但它又承担着补 coverage 的职责，所以它不是简单的一刀切问题，而是 Method2 现有构场逻辑内部的硬矛盾。

### 5.10 实验 10：锚点底图加残差实验

第十个实验的目标，是验证“锚点底图 + 细节残差”这个拆分思路是否成立。

实现逻辑是，先用锚点底图生成一版面级场，再把当前目标场减去这版底图，得到残差场；分别看底图本身的质量，以及“把残差作为目标场重新积分”会发生什么。

结果是，底图本身非常健康，但残差场的旋度和冲突非常强，一旦把它单独塞给积分器，结果反而非常差。

初步结论是，真正有毒的部分不是底图，而是残差；问题不在“有没有 base”，而在“细节是以什么方式叠加到 base 上”。

### 5.11 实验 11：构场器变体实验

第十一个实验的目标，是比较“当前目标场”和“更忠于原始采样的 sample-fit 场”到底谁更合理。

实现逻辑是，测试了纯 sample-fit、对 sample-fit 加锚点先验、只在四采样面上替换、对三采样面做带锚点的折中等多种构场变体。

结果很有代表性：越是让目标场忠于原始采样，它在解释样本这件事上越好，但全局旋度和最终求解效果往往越差；尤其是三采样面的不稳定性，会迅速把这种局部忠实变成全局冲突。

初步结论是，问题不是“当前场不够贴采样，所以只要更贴就好”，恰恰相反，未经处理的 sample-fit 场往往更危险。

### 5.12 实验 12：绝对场投影实验

第十二个实验的目标，是在送入求解器之前先做一次绝对场空间的“排毒”。

实现逻辑是，把当前场或者 sample-fit 场，用锚点底图作为基底，做一次带置信度门控和旋度正则的绝对场投影。高置信区域尽量保留，低置信区域向锚点底图回退，同时惩罚相邻面之间的场跳变。

结果是，旋度确实明显降下来了，场看起来也更平顺了，但它往往会过度向锚点底图塌缩，造成细节损失，甚至把问题从“局部冲突”转移成“整体过平”和新的全局漂移。

初步结论是，单纯在绝对场上做 L2 式平滑投影，会把好细节和坏毒素一起抹掉。

### 5.13 实验 13：残差场投影实验

第十三个实验的目标，是验证“只在相对锚点底图的残差空间里做投影”是否比实验 12 更稳。

实现逻辑是，先把数据场减去锚点底图，只保留细节残差，然后在这个残差场上做置信度门控和旋度正则，再把投影后的残差加回锚点底图。这里测试了来自当前场和 sample-fit 场的多种残差版本，也比较了软门控、严格门控和不同旋度权重。

结果是，这一组里终于出现了一个相对最优的折中版本：它既比原始当前场更干净，也不像实验 12 那样一股脑塌回锚点底图，后来我们把这版结果当成 Method2.5 的上游字段来源。

初步结论是，场空间投影是有价值的，但它只能把问题从“非常脏”变成“相对可控”，还没有从根本上解决高失配资产上的几何可行性问题。

## 6. 到实验 13 为止的阶段性推理链

走完前十三个实验之后，逻辑已经比较清楚了：

1. Method2 的主问题不是单一某个参数，也不是线性求解器本身，而是上游目标 Jacobian 场的构造带着系统性的混合冲突和旋度。
2. 这个冲突不是局部面片完全不可解，而是许多局部 patch 可以解释，但一旦全局拼装、叠加 fallback、跨来源混合，再进入一个连续 UV 参数化，就会积累成全局矛盾。
3. 锚点底图本身是相对健康的，真正有毒的是“想把高模细节直接注入低模”的那部分残差。
4. 这也是为什么“更贴采样”不等于“更可解”。
5. 实验 13 说明场空间投影是有价值的，但它不是最终解，只是把问题清理到了一个更适合继续研究的状态。

## 7. 后续扩展验证

### 7.1 Route B：求解空间边界约束

在前十三个实验之后，我们怀疑是不是 Method2 的线性积分器太自由，才会让越界和漂移失控。

Route B 的实现逻辑，是保留 Method2 的上游构场，只在下游求解阶段增加更硬的盒约束和 barrier，强行限制 UV 不要飞出安全区域。

结果证明，这类约束确实能压住越界，但压不住拉伸和翻转。很多时候它只是把原本往外泄的误差重新挤回内部。

初步结论是，越界是症状之一，但不是根病本身。

### 7.2 Route C：拓扑释放

接着我们又试了 Route C，也就是给系统更多拓扑自由度，主动切缝，释放局部应力。

这个方向的出发点是：如果连续参数化本身扛不住那些局部冲突，也许应该让系统在该断开的地方断开。

结果表明，切缝确实能在一些区域释放冲突，也能改变越界和部分拉伸的分布，但它并不会自动带来全面改善；有些场在切开之后，反而把原本被平滑掩盖的尺度冲突暴露得更彻底。

初步结论是，拓扑释放是一种真实自由度，但它仍然只能解决问题的一部分。

### 7.3 Method2.5：干净场加非线性求解器

在实验 13 拿到一版相对最干净的残差投影场之后，我们又做了一个关键的组合实验，也就是后来定义的 Method2.5。

它的逻辑很简单：

1. 沿用 Method2 的 correspondence 和内部状态。
2. 用实验 13 最好的残差投影场替换掉原始目标场。
3. 先线性积分出一个更干净的初始化。
4. 最后交给 Method4 的非线性 barrier 求解器。

这个实验的意义，不是做一个新的默认方法，而是回答一个更明确的问题：如果把更干净的场喂给更强的求解器，问题会不会显著缓解。

结果是：

1. 在像 `corgi` 这样高低模失配较小的资产上，Method2.5 是成立的。更干净的场确实能让 Method4 从“根本走不动”变成“可以优化并得到更好的结果”。
2. 但在 `massive` 这种高失配、重简化、低锚点、多分块的资产上，Method2.5 只能保住更好的重投影，没法自动兑现成更好的几何可行性。

初步结论是，Method2.5 证明了“干净场加更强求解器”在低失配条件下是通的，但在高失配资产上，低模承载能力本身已经成为主导瓶颈。

## 8. 当前阶段的总判断

到目前为止，我们已经可以把问题分成两层。

第一层，是 Method2 自己的工程病灶，也就是上游构场存在明显的多源混合、fallback 污染和可积性缺陷。这一层病灶，通过前十三个实验和实验 13 的 projector，已经被相当明确地定位了。

第二层，是更高层的承载能力问题：当高低模失配足够大、低模自由度不足、拓扑分块复杂、锚点又很稀时，即便我们已经把目标场清理得比原来干净，低模也未必还能承受那份高模细节要求。

这也是为什么：

1. 在 `corgi` 上，问题主要表现为“脏场拖垮求解器”。
2. 在 `massive` 上，问题更像“低模本身装不下这份场”。

## 9. 组会可落的结论

如果用一句话总结这十几个实验，我们最终得到的不是某一个神奇改法，而是一张逐层收敛的诊断图。

Method2 的确先有构场问题，Method4 也的确能在一定程度上修几何，但真正困难的资产上，问题已经不再只是“场不够干净”或“求解器不够强”，而是任务本身触到了低模自由度和连续参数化假设的边界。

也就是说，这十几个实验的最大价值，不是把一个方法调通了，而是把“哪里还能继续改、哪里已经不是调参问题”这件事讲清楚了。
